# Блогикум

Блог-платформа на Django 3.2. Пользователи могут регистрироваться, публиковать записи с изображениями, оставлять комментарии и управлять своим профилем. Администратор настраивает категории и местоположения через панель Django Admin.

## Структура проекта

```
django_sprint4/
├── blogicum/              # Django-проект
│   ├── blog/              # Приложение блога (посты, комментарии, профили)
│   ├── pages/             # Статичные страницы и обработчики ошибок
│   ├── templates/         # HTML-шаблоны
│   └── manage.py
├── html/                  # Статические файлы (CSS, изображения)
├── tests/                 # Автотесты Практикума
├── db.json                # Фикстуры (опционально)
├── requirements.txt
└── venv/                  # Виртуальное окружение
```

## Установка и запуск

```bash
cd django_sprint4
python -m venv venv

# Windows
venv\Scripts\activate

# Linux / macOS
source venv/bin/activate

pip install -r requirements.txt
# На Python 3.12 при ошибке установки Pillow:
pip install "Pillow>=10.0.0"

cd blogicum
python manage.py migrate
python manage.py runserver
```

Сайт будет доступен по адресу http://127.0.0.1:8000/

Загрузка демо-данных (необязательно):

```bash
python manage.py loaddata ../db.json
```

Запуск тестов:

```bash
pytest
```

## Основные возможности

### Публикации

- Лента записей на главной странице с пагинацией (10 постов на страницу).
- Просмотр поста по адресу `/posts/<id>/`.
- Фильтрация по категории: `/category/<slug>/`.
- Прикрепление изображения к публикации.
- Отложенные публикации: пост с датой в будущем скрыт от всех, кроме автора.
- Снятые с публикации посты и посты в неопубликованных категориях видны только автору.

### Пользователи

- Регистрация: `/auth/registration/`
- Вход / выход: `/auth/login/`, `/auth/logout/`
- Профиль: `/profile/<username>/`
- Редактирование профиля (имя, фамилия, логин, email): `/edit_profile/`
- Смена и восстановление пароля через стандартные маршруты Django Auth (`/auth/password_change/`, `/auth/password_reset/` и др.)

### Комментарии

- Добавление комментария к опубликованному посту (только авторизованные пользователи).
- Редактирование и удаление собственных комментариев.
- Сортировка комментариев от старых к новым.
- Счётчик комментариев в карточках постов на главной, в профиле и в категории.

### Администрирование

- Категории и местоположения создаёт только администратор через `/admin/`.
- Публикации и комментарии пользователи создают самостоятельно через интерфейс сайта.

### Прочее

- Кастомные страницы ошибок: 403 (CSRF), 404, 500.
- Статичные страницы «О проекте» и «Наши правила» на CBV.
- Файловый бэкенд e-mail: письма сохраняются в `blogicum/sent_emails/`.

## Модели (`blog/models.py`)

| Модель | Назначение |
|--------|------------|
| `Category` | Категория публикации (заголовок, slug, описание, флаг публикации) |
| `Location` | Местоположение автора (необязательное поле поста) |
| `Post` | Публикация: заголовок, текст, дата, автор, категория, место, изображение |
| `Comment` | Комментарий к посту: текст, автор, дата создания |

---

## Формы (`blog/forms.py`)

| Форма | Поля | Назначение |
|-------|------|------------|
| `PostForm` | title, text, pub_date, location, category, image | Создание и редактирование поста |
| `CommentForm` | text | Создание и редактирование комментария |
| `UserProfileForm` | first_name, last_name, username, email | Редактирование профиля |
| `RegistrationForm` | first_name, last_name, username, email, password1, password2 | Регистрация нового пользователя |

## Представления (`blog/views.py`)

### Вспомогательные функции

| Функция | Описание |
|---------|----------|
| `get_published_posts(queryset)` | Возвращает queryset опубликованных постов: `is_published=True`, дата не в будущем, категория опубликована |
| `annotate_comment_count(queryset)` | Добавляет поле `comment_count` и сортирует посты по дате публикации (новые первыми) |
| `paginate_posts(request, queryset)` | Разбивает queryset на страницы по 10 записей, возвращает объект `page_obj` |
| `get_post_for_detail(post_id, user)` | Возвращает пост для страницы просмотра: автор видит свои скрытые/отложенные посты, остальные — только опубликованные |

### Публичные страницы

| Функция | URL | Описание |
|---------|-----|----------|
| `index` | `/` | Главная: лента опубликованных постов с пагинацией |
| `post_detail` | `/posts/<id>/` | Страница поста, список комментариев и форма добавления комментария |
| `category_posts` | `/category/<slug>/` | Посты выбранной опубликованной категории |
| `profile` | `/profile/<username>/` | Профиль пользователя и его публикации; автор видит все свои посты, гости — только опубликованные |
| `registration` | `/auth/registration/` | Регистрация нового пользователя |

### Требуют авторизации (`@login_required`)

| Функция | URL | Описание |
|---------|-----|----------|
| `own_profile` | `/profile/` | Перенаправляет на страницу профиля текущего пользователя (используется после входа) |
| `edit_profile` | `/edit_profile/` | Редактирование имени, фамилии, логина и e-mail |
| `create_post` | `/posts/create/` | Создание новой публикации; после сохранения — редирект в профиль |
| `edit_post` | `/posts/<post_id>/edit/` | Редактирование поста (только автор); чужим пользователям — редирект на просмотр |
| `delete_post` | `/posts/<post_id>/delete/` | Подтверждение и удаление поста (только автор); редирект на главную |
| `add_comment` | `/posts/<post_id>/comment/` | Добавление комментария к опубликованному посту |
| `edit_comment` | `/posts/<post_id>/edit_comment/<comment_id>/` | Редактирование своего комментария |
| `delete_comment` | `/posts/<post_id>/delete_comment/<comment_id>/` | Подтверждение и удаление своего комментария |

## Приложение `pages`

| Компонент | Описание |
|-----------|----------|
| `AboutView` (CBV) | Статичная страница «О проекте» — `/pages/about/` |

| `csrf_failure` | Обработчик ошибки 403 CSRF (`CSRF_FAILURE_VIEW`) |
| `server_error` | Обработчик ошибки 500 (`handler500`) |

---

## Маршруты аутентификации (`/auth/`)

Подключены стандартные маршруты `django.contrib.auth.urls`:

| URL | Назначение |
|-----|------------|
| `/auth/login/` | Вход |
| `/auth/logout/` | Выход |
| `/auth/password_change/` | Смена пароля |
| `/auth/password_reset/` | Восстановление пароля |

После входа пользователь перенаправляется на `/profile/` (далее — на свой профиль). После выхода — на главную страницу.
